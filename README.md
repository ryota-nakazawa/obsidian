# 外部脳 vault

このvaultは、外部ソース、整理済み知識、日々の記録、業務手順、質問回答をつなぐための個人知識グラフです。

## 使うフォルダ

| フォルダ | 役割 |
|---|---|
| `raw/` | 外部ソース置き場。記事、資料、Webクリップ、PDFのテキスト化など |
| `wiki/` | 整理済み知識。ソース要約、概念ページ、関連付け済みノート |
| `daily/` | 日々の記録。作業ログ、日記、感情メモ、気づき |
| `ops/` | 再利用する手順。仕事・生活のやり方、チェックリスト |
| `qa/` | Codexへの質問と再利用価値のある回答 |
| `inbox/` | 未整理メモの一時置き場 |
| `templates/` | ノートの型 |
| `.codex/commands/` | Codexに頼む定型作業 |

## 基本の流れ

1. 外部記事や資料を `raw/` に入れる。
2. Codexに頼んで `wiki/` に要約・概念・関連リンクを作る。
3. 日々の作業や感情は `daily/` に書く。
4. 繰り返す作業は `ops/` に手順化する。
5. 重要な質問回答は `qa/` に保存する。
6. 重要な概念、手順、プロジェクトだけ `[[wikilink]]` でつなぐ。

## Codexへの依頼例

```text
rawに追加した記事をwiki化して
```

```text
今日のdailyに重要なwikilinkだけ貼って
```

```text
今週のdailyとqaを読んで週報を書いて
```

```text
この回答をqaに保存して、関連するwikiやopsにリンクして
```

## 方針

- まずは日本語で読みやすく書く。
- 完璧な分類より、後からつなげられることを優先する。
- `raw/` は原本、`wiki/` は解釈・整理、`daily/` は体験、`ops/` は手順、`qa/` は回答として分ける。
- リンクを貼りすぎない。あとで探したいものだけつなぐ。
