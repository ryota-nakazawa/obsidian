# AIエージェント評価の極意（note・Anthropic記事解説）

## 要約（約200字）

Anthropicの「AIエージェント評価」記事の解説。AIは[[非決定性]]と複雑性を持つため従来テストでは不十分とし、[[タスク]]・[[トライアル]]・[[グレーダー]]・[[評価ハーネス]]という構成要素を学校のテストに例えて整理する。[[コードベースグレーダー]]・[[モデルベースグレーダー]]・[[人間による評価]]の使い分け、到達可能性を測る[[Pass@K]]と信頼性を測る[[Pass＾K]]の区別、エージェント種別ごとの評価軸を示し、[[失敗事例からの評価構築]]（20〜50件から開始）という実践手順を提示する。

## 主要概念

- [[非決定性]]
- [[タスク]] / [[トライアル]] / [[トランスクリプト]] / [[アウトカム]]
- [[グレーダー]]
  - [[コードベースグレーダー]]
  - [[モデルベースグレーダー]]（LLM判定）
  - [[人間による評価]]
- [[評価ハーネス]]
- [[Pass@K]]（到達可能性）
- [[Pass＾K]]（信頼性）※本来の表記は Pass^K（ファイル名制約のため全角＾）
- [[エージェント種別ごとの評価]]
  - コーディングエージェント → [[ユニットテスト]]
  - 会話エージェント → 完了率・ターン数
  - リサーチエージェント → 根拠と網羅性
- [[失敗事例からの評価構築]]
- [[環境の分離]]（クリーンなテスト環境）
- [[部分点]]
- [[外部ベンチマーク]]（SWE-bench、WebArena）

## 他のソースとのつながり

- [[AI Evals 評価設計（Arpable）]] は本記事の一次情報（Anthropicのevals記事）を引用しており、同じ [[グレーダー]]・[[評価ハーネス]] の枠組みを企業導入の文脈（つなぐ→守る→測る）に位置づけ直している。
- [[Evaluation Best Practices（OpenAI）]] も [[非決定性]] を出発点とする点、[[グレーダー]] を「メトリクス・LLM判定・人間」の3種で組み合わせる点で共通。ただし本記事の [[Pass@K]] / [[Pass＾K]] という試行回数ベースの指標はOpenAIガイドには登場せず、本ノート固有の貢献。
- 「失敗事例20〜50件から始める」アプローチは、OpenAIの [[Production Flywheel]]（本番の失敗を評価ケースへ戻す循環）の初期ブートストラップ版と読める。
